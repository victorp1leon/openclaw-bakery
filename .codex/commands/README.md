# Local Speckit Commands

These local command definitions support Spec-Driven workflow in canonical paths under `documentation/specs`.

## Commands
- `speckit.index`: quick open/read of canonical specs index.
- `speckit.template`: quick open/read of feature package template.
- `speckit.manifest`: quick open/read of migration manifest.
- `speckit.validate`: structure and link sanity checks for canonical specs paths.

## Path Policy
- Canonical specs root: `documentation/specs`
- Sessions remain in: `documentation/ai_collaboration/plans/*/sessions`
- Contracts migration target: `documentation/specs/contracts/components`
